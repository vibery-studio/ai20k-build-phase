Pattern 01

Chat + context panel

Trợ giảng AI & Q&A bài giảng Gia sư Socratic Tra cứu quy định & admissions LMS chatbot có memory

User hỏi chat, agent trả lời. Panel bên cạnh show context — source docs, progress, reasoning.

Pattern 02

Upload → dashboard

Phân tích feedback SV OCR bài viết tay Ảnh kính hiển vi / Medical VLM Rà soát CAD / bản vẽ Số hóa bảng trắng

User upload file (CSV, audio, PDF), agent xử lý batch, trả về dashboard — aggregate stats + top quotes + trend chart.

Key UX decision Cho user xem raw data bao nhiêu? — hide hết = user không tin. Show hết = overwhelm. Middle: show 3-5 examples + aggregate stats. User drill-down khi cần.

UI pattern · 04 / 07
Pattern 04

Wizard + inline audit

Soạn syllabus / ABET Soạn giáo án GDPT 2018 Tạo đề thi Bloom Viết nhận xét học bạ Thiết kế CTĐT

Guided multi-step creation với compliance check. User điền từng bước, agent audit inline — không chờ submit.

Key UX decision Audit errors block flow hay chỉ highlight? — Compliance cao (accreditation, legal): must-block. Compliance vừa (teacher judgment, draft): highlight + override. Chọn sai = user chạy vòng hoặc bỏ dùng.

UI pattern · 05 / 07
Pattern 05

Draft → approve → send

Truyền thông khẩn cấp Giao tiếp phụ huynh Xử lý đơn từ tự động Onboarding nhân viên Social commerce agent

Irreversible action (send tin qua Zalo / SMS / Email, push notification). Cần double-confirmation UI — không có "Ctrl-Z" trong đời thật.

Key UX decision Confirmation UX — double-tap? type 'CONFIRM'? preview per channel? SMS preview ≠ Email preview ≠ Push preview. Mỗi channel = 1 preview card, user scan nhanh.

Pattern 06 · Interactive walkthrough

Queue + approval dashboard

Moderation cộng đồng Chấm bài quy mô lớn Cảnh báo sớm SV Rà soát hợp đồng / chính sách Liêm chính học thuật

Agent classify hàng loạt, human review sau. Design cho throughput. Không build live — có interactive ref link dưới đây.

Real-time streaming

Luyện phỏng vấn / avatar AI Huấn luyện thể thao qua video Q&A hội thảo voice Thí nghiệm ảo multi-agent Hệ thống luyện tiếng Anh

T · Transparency Deep demo · 10 phút

Panel = cửa sổ vào AI brain

Thêm Sources panel (3 refs) + Confidence bar. User nhìn 1 phát là biết AI dựa vào đâu.

[T] Add transparency: for each AI response, populate right panel with "Sources" section (list 2-3 doc refs the AI claims to use) and "Confidence" section (High/Medium/Low badge + bar).
Why Transparency không phải show hết. Show đủ để user có action. Source user click được = trust. Chain-of-thought dump = overwhelm. Panel = chỗ show những gì user actionable.
17 / 22
C · Control Deep demo · 10 phút

Câu trả lời chưa chính xác? User có đường sửa

"Flag" link dưới mỗi câu + correction flow + Stop button giữa khi AI đang stream.

[C] Add control: "Flag as inaccurate" link below each AI response (opens correction form: what was wrong + what should it say), "Stop" button while streaming, pin-a-source from panel so AI prioritizes it next turn.
Why flag, not just thumbs-down Flag = data loop, không phải courtesy. Mỗi flag + correction = training signal. Thumbs-down = bạn mất user, không lấy được gì. Correction form biến 1 user angry thành 1 data point giá trị.
18 / 22
R · Recovery Deep demo · 8 phút

Recovery = pre-flight, không chỉ post-hoc catch

Retry button per message + pre-flight validation (input <5 chars warning, vague question → suggest rephrase).

[R] Add recovery: "Retry" button on each AI message (regenerate with same context), pre-flight check — if user input < 5 chars show warning "Câu hỏi quá ngắn, có thể AI hiểu lệch", suggest clearer phrasing.
Why pre-flight Bug cheapest to catch là bug chưa xảy ra. User gõ "aaa" — không cần call LLM, cảnh báo trước. Tiết kiệm cost + user không cần đọc câu trả lời rác. Post-hoc retry vẫn giữ, nhưng pre-flight chặn 60% lỗi dễ.
